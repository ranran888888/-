# 评分系统研究：mod1 打新猎手 + mod5 KOL雷达 模块化实施方案（v1.1对齐版）

## 1. 目标与边界

### 1.1 目标
- 将 **mod1 打新猎手** 独立为“早期 Alpha 精准捕获模块”。
- 将 **mod5 KOL 雷达** 独立为“可插拔通用能力模块（万能插槽）”，可服务 mod1，也可服务 mod2/mod3/mod4。
- 严格保留 v1.1 的统一评分主线：
  `final = base × source × market × cross × decay × propagation`

### 1.2 不变项（硬约束）
- 不新增第六模块，不改变五大模块结构。
- 不改变统一评分公式与字段命名。
- 不改变 base/source/market/cross/decay/propagation 因子定义。
- 不引入未审批“自动学习/自动调参”机制。

### 1.3 本方案解决的核心痛点
- 痛点A：早期高价值信号常来自低粉、未知账号，传统 source 权重会把它们压入 archive。
- 痛点B：只靠“赛道名词关键词”会漏掉“新赛道第一条信号”。
- 痛点C：各业务模块都依赖 KOL 信息，但依赖关系在工程上不显式，复用成本高。

## 2. mod1 打新猎手：从“关键词筛子”升级为“早期 Alpha 捕获器”

### 2.1 双层采集入口（保留 v1.1 架构，不改评分公式）

#### 第一层（P1）：赛道直接信号词（高频）
- 作用：快速把“明确发行事件”送入流水线。
- 关键词扩展方向：mint/铸造/铭文/打钱/social experiment/new protocol 等中英文直信号。

#### 第二层（P13/P14）：Builder 行为关键词（中频）
- 作用：捕获“你还不知道名字的新项目”。
- 结构：五层三档（动作/技术栈/价值主张/成熟度/证据）。
- 输出：`builder_action_layers_hit`（0~5层命中）作为可观测输入。

### 2.2 mod1 内部评分精准化（不改五维总权重）
- 保持 base 五维：novelty(30)/verifiability(20)/narrative(20)/innovation(20)/timeliness(10)。
- innovation 维度内部采用双子维：
  - A：创新关键词命中（0~10）
  - B：行为层数命中（0~10）
- 总 innovation 仍封顶 20，不突破 v1.1。

### 2.3 早期 Alpha 信号“提权但不失控”策略（方向2落地）

仅在 mod1 且满足条件时，对 source 进行**下限保护**（floor），避免高质量早期信号被数学锁死：

- 输入证据1：`builder_action_layers_hit`
- 输入证据2：`notable_follower_count / s_tier_follower_count`（由 mod5 提供）

建议规则（可配置）：
- `layers_hit >= 3` 且 `(s_tier >= 2 或 notable >= 5)` → source floor 0.7
- `layers_hit >= 3` 且 `notable >= 2` → source floor 0.6
- `layers_hit >= 4`（无社交验证）→ source floor 0.55
- 其余情况不提权

> 解释：不改变 source 因子语义框架，只在 mod1 场景引入“内容+社交图谱”双证据防误报提权，降低漏掉早期 Alpha 的概率。

### 2.4 冷却复推策略（P18）
- 从“与上次分数比较”改为“与初始入库分比较”：
  `re_push_trigger = new_score - initial_score >= threshold`
- 价值：让“逐步升温”的信号能被重新推送，不被分步小增量卡死。

## 3. mod5 KOL 雷达：从“单点评估器”升级为“通用可插拔能力层”

### 3.1 双角色定位（不新增模块，只显式化职责）

#### 角色A：业务模块（mod5 自身）
- 继续执行 KOL 发现、筛选、确认、评估、更新。

#### 角色B：服务能力层（对其它模块提供接口）
- 为 mod1/mod2/mod3/mod4 提供标准 KOL 能力查询。

### 3.2 标准接口（万能插槽）

建议统一接口（读接口优先）：
- `get_kol_tier(author_id)` → S/A/B/C/unknown
- `get_source_type(author_id)` → twitter_kol_s/.../twitter_unknown
- `check_notable_followers(author_id)` → notable/s_tier/a_tier counts
- `get_kol_track_record(author_id)` → 命中率、准确率、偏好领域

写接口（mod5 内部流程）：
- `discover_kol_multi_channel(...)`
- `scan_kol_following(...)`
- `confirm_kol_candidates(...)`
- `update_kol_registry(...)`

### 3.3 mod5 核心能力保留并强化
- 保留五维评分：lead_time/hit_rate/accuracy/independence/consistency。
- 保留窗口过滤、清洗、去重、样本门槛、紧急模式、失败模式回放。
- 保留趋势聚类 6h 报告：
  `48h内≥3独立来源 → 趋势信号 → 新概念 → 升温 → 推送报告`

---

## 4. 两模块协同数据流（落地版）

1. 采集层输出 raw_event（含 P1/P13 命中信息）
2. enricher 调用 mod5 只读接口补齐：
   - source_type（KOL分层）
   - notable_followers 证据
3. mod1 计算 base 与 explain，并输出 registry_updates
4. scorer 统一计算 final（公式不变）
5. 推送矩阵决策（immediate/digest/archive）
6. mod5 周期评估 KOL 并更新 registry，反哺所有模块

---

## 5. 验收标准（可测试、可回放、可插拔）

### 5.1 mod1 验收
- 能捕获未知赛道第一条信号（行为关键词路径）。
- 能区分高质量早期信号与垃圾噪音（双证据提权后仍可拦截低 Base）。
- 输出完整可回放字段：input/hit_rules/base_breakdown/factors/final/observability。

### 5.2 mod5 验收
- 可独立运行 KOL 发现/筛选/确认/评估/更新。
- 可被其它模块直接调用标准只读接口。
- 失败模式与紧急模式可回放。

### 5.3 系统级验收
- 统一评分主线与字段全量存在。
- cross 封顶规则稳定。
- 不依赖飞书要素（去飞书化一致）。

---

## 6. 分阶段实施计划（高可行性）

### Phase 1（先可用）
- 固化 mod1 + mod5 当前能力。
- 打通 mod1 调用 mod5 只读接口。
- 跑历史样本回放（ORDI/ETHS/BOME/PEOPLE 等）。

### Phase 2（提精准）
- 上线“层数+社交图谱”双证据提权（仅 mod1）。
- 启用 P18 复推与趋势6h报告联动。

### Phase 3（全模块复用）
- mod2/mod3/mod4 统一改为调用 mod5 标准接口，不复制 KOL 判断逻辑。
- 完成跨模块一致性回归测试。

---

## 7. 最终结论

该方案在不破坏 v1.1 硬约束的前提下，实现：
- **mod1 更像“早期 Alpha 捕获器”**：减少“新项目低粉首发被压死”的漏报。
- **mod5 更像“通用 KOL 能力底座”**：可插 mod1，也可插其它模块。
- **公式与架构保持稳定**：不改统一评分主线、不改五大模块框架，工程可实施性高。
